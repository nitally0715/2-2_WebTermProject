### 3. 화면 흐름

	- 화면1.1 (로그인) :

		2. ID, PASSWORD 입력창

		4. Register 버튼 : 화면1.2
	
	- 화면1.2 (회원가입) :
		1. ID 입력창
		2. PASSWORD 입력창

	- 화면2 (메인) :

		3. 게임 메뉴 (타자연습, 퀴즈, 검색) : 클릭 시 해당 메뉴의 게임화면으로 이동.

### 3. 배포
	- Docker와 Docker compose를 사용하여 로컬 환경에서 돌아갈 수 있게끔
	
### 4. DB

	- mySTL을 사용
	- 사용자 :
		1. ID
		2. PASSWORD
		3. 누적타자수
		4. 누적퀴즈수
		5. 레벨
		6. 최고타수
	- 타자연습 문제 :
		1. 사용 언어
		2. 연습문장
	- 퀴즈 문제 : 
		1. 사용 언어
		2. 함수 또는 키워드
		3. 2.에 대한 설명
### 5. CRITICAL RULES

	- DB 삭제 명령 (DROP, TRUNCATE 등) 절대 금지
	- 사용자 정보 로그 출력 금지
	- eval(), system() 등의 동적 실행 금지
	- 기존 기능 제거 시 반드시 사전 승인 필요
